# 翼家校园通信系统最终项目文档

## 1. 问题定义

### 1.1 项目名称
翼家校园通信系统

### 1.2 项目背景
随着校园信息化建设不断推进，学生与教师在课程交流、资料共享、群组讨论和通知发布等场景下，对即时通信工具的需求越来越明确。通用社交软件虽然可以满足基础聊天需求，但在课程设计展示、本地部署演示、管理员广播和校园账号管理等方面缺乏针对性。因此，本项目面向课程设计场景，开发一个可本地运行、可打包交付、具备校园通信核心能力的桌面应用系统。

### 1.3 项目目标
本项目目标是实现一个具备用户注册登录、好友管理、单聊群聊、消息记录、文件传输、在线状态展示与管理员广播能力的校园通信系统，并最终以 Windows 桌面应用形式交付，满足课程设计提交与答辩展示需求。

## 2. 需求分析

### 2.1 功能需求
1. 支持用户注册、登录和退出。
2. 支持通过校园号、手机号、邮箱搜索用户。
3. 支持好友申请、好友审批与好友备注。
4. 支持单聊与群聊。
5. 支持消息发送、历史消息查看与消息已读。
6. 支持文件上传、发送与下载。
7. 支持在线状态展示。
8. 支持管理员登录、系统概况查看与广播通知。
9. 支持打包为可直接运行的 Windows 桌面应用。

### 2.2 非功能需求
1. 界面风格统一，适合课程设计展示。
2. 系统结构清晰，便于维护与讲解。
3. 支持本地部署运行，适合无外网环境演示。
4. 支持本地持久化保存用户、消息和聊天记录。
5. 交付形式以 `.exe` 桌面应用为主，不再以网页独立交付为重点。

### 2.3 最终修订后的需求说明
在项目后期结合实际开发与交付要求，对前期需求进行了调整：
1. 系统最终以桌面应用加内置后端的方式交付。
2. 系统名称统一为“翼家校园通信系统”。
3. 前端整体视觉主题统一调整为蓝色和绿色。
4. 管理员端采用独立入口，与普通用户端分离。
5. 删除文档中以网页单独交付为主的描述，统一为桌面应用交付。

## 3. 总体设计

### 3.1 系统架构
系统采用前后端分离并结合桌面封装的总体架构：
- 前端：React + TypeScript + Material UI
- 后端：Spring Boot
- 数据存储：H2 本地文件数据库
- 实时通信：WebSocket / STOMP
- 桌面封装：Electron
- 打包与启动：PowerShell 脚本 + jpackage + C# 启动器

系统运行时，桌面程序先启动本地内置后端，再加载前端登录界面，从而形成完整的本地桌面应用体验。

### 3.2 功能模块划分
1. 用户认证模块
2. 用户与好友管理模块
3. 即时通信模块
4. 群聊管理模块
5. 文件传输模块
6. 在线状态模块
7. 管理员模块
8. 桌面封装与部署模块

## 4. 详细设计

### 4.1 用户认证模块
用户认证模块负责普通用户注册、登录、退出和当前用户信息获取。后端基于 Spring Security 与 JWT 机制完成身份验证与访问控制。

### 4.2 好友管理模块
好友管理模块负责用户搜索、好友申请、好友审批、好友备注和联系人列表维护，是校园交流网络建立的基础。

### 4.3 即时通信模块
即时通信模块负责单聊、群聊、消息发送、消息记录查看与消息已读功能，并通过 WebSocket 实现消息实时同步。

### 4.4 文件传输模块
文件传输模块支持聊天中的文件上传、发送和下载，后端保存文件并在消息记录中保留文件信息。

### 4.5 在线状态模块
系统通过 WebSocket 会话状态维护在线用户信息，用户可查看联系人当前是否在线，提高沟通效率。

### 4.6 管理员模块
管理员模块包括管理员登录、系统概况查看、在线用户查看和广播消息发送，满足课程设计展示和管理需求。

### 4.7 桌面应用模块
为了满足最终交付要求，系统增加了桌面端封装模块，主要包括：
1. 内置后端自动启动。
2. 桌面窗口加载登录页。
3. 生成单文件 `.exe`。
4. 生成安装包版本。
